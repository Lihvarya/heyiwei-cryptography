## 📜 项目简介 (Introduction)

本项目是「何意味」主题的趣味文本编码工具。它将任意文本（UTF-8 编码）转化为一系列读音相似、但意义荒谬的中文词组序列，例如：

> **原始文本:** `Hello World!`
> **加密结果:** `核疫卫何意味喝益胃何异味河移位何以畏河移位河易危河异味河易危何意味何以慰`

本工具基于 `BigInt` 实现，确保可以对任意长度的文本进行高精度和无损的编码与解码。


## ✨ 主要功能 (Features)

*   **单文件运行:** 仅需一个 `何意味.html` 文件，下载后即可在任意现代浏览器中运行，无需服务器或后端依赖。
*   **高精度编码:** 使用 JavaScript `BigInt` 类型处理整个文本字节序列，确保无论输入文本多长，都能进行准确的 Base-N 转换，无溢出风险。
*   **自定义 Base-30 体系:** 利用代码中定义的 谐音词组（`PHRASES`）作为编码的基数（Base）。
*   **双向转换:** 提供**加密**（文本 -> 谐音词组）和**解密**（谐音词组 -> 文本）功能。
*   **优化 UI:** 简洁的 Material 设计风格，支持输入清空、粘贴和结果复制功能，提升用户体验。

## ⚙️ 编码机制 (Mechanism)

「何意味生成器」的核心是一个自定义的 **Base-N 编码**：

1.  **数据转换:** 输入的 UTF-8 文本首先被 `TextEncoder` 转换为一个字节数组。
2.  **BigInt 整合:** 整个字节数组被视为一个巨大的二进制数，并使用 `BigInt` 将其解析为一个单一的大整数 (`largeNumber`)。
3.  **基数转换:** 我们将这个大整数从 Base-256 (字节) 转换到自定义的 **Base-30**。
4.  **字符映射:** 转换过程中产生的每个数字余数 (0 到 29) 被映射到 `PHRASES` 数组中对应的谐音词组，从而生成最终的「何意味」序列。

由于词组长度不一，解密时采用了贪婪匹配算法，优先匹配长度最长的词组以确保解析的准确性。

## 🚀 如何使用 (Usage)

本项目是完全前端的，无需安装任何依赖。

1.  **下载文件:** 克隆本仓库或直接下载 `何意味.html` 文件。
2.  **打开浏览器:** 使用任意现代浏览器（Chrome, Firefox, Edge 等）打开 `何意味.html` 文件。
3.  **输入文本:** 在输入框中输入你想要编码或解码的文本。
4.  **点击按钮:** 选择 **加密** 或 **解密** 按钮，结果将立即显示在下方。
